# Hannah Personal Site Roadmap

## Product Vision

Build a personal website that helps people understand Hannah as an emerging AI product manager: curious, rigorous, design-aware, and willing to build in public.

The site should not become a generic resume page. It should become a living portfolio for AI product thinking, AI coding experiments, learning notes, and future product demos.

## Current Product Principles

- Be true before being polished.
- Use the website as a product lab, not a decoration project.
- Keep the "window" concept as a creative direction, but validate the interaction before committing to it.
- Every public section should answer a visitor's real question.
- Ship small versions and review them honestly.

## Milestones

### v0.1 Publish the First Site

Status: Done

Goal: Put the first static website online and make the project trackable through Git.

Scope:

- Create static HTML/CSS/JS site.
- Add visual hero asset.
- Initialize Git repository.
- Push to GitHub.
- Publish with GitHub Pages.

Definition of Done:

- The repository exists on GitHub.
- The site is accessible through a public GitHub Pages URL.
- Local `main` branch tracks remote `origin/main`.

Live URL:

- https://hannahhan1625.github.io/Hannah.github.io/

### v0.2 Replace Placeholders With Real Identity

Status: Next

Goal: Make the homepage truthful enough to share with a real person.

Scope:

- Decide the public display name.
- Replace placeholder email and social links.
- Write a concise personal positioning statement.
- Write a short transition story from project management, operations, and design into AI product work.
- Clarify the audience: AI builders, product teams, founders, researchers, or peers.

Definition of Done:

- No placeholder contact links remain.
- The first screen explains who Hannah is becoming.
- A visitor can understand why they should keep reading within 10 seconds.

### v0.3 Design the Window Concept

Status: Planned

Goal: Turn the "window" idea into a clear interaction model without losing usability.

Scope:

- Define what each window represents.
- Decide whether windows are navigation cards, project previews, modal dialogs, or full pages.
- Create one low-fidelity version first.
- Compare it against the current portfolio layout.

Definition of Done:

- The window concept has a written interaction spec.
- The homepage has one tested prototype direction.
- The team decision is recorded: keep, revise, or pause the window concept.

### v0.4 Add the First Real Project Case

Status: Planned

Goal: Add one project case that proves AI product thinking, not just taste.

Scope:

- Choose one AI coding product or learning artifact.
- Write the user problem.
- Explain the workflow and tools.
- Add screenshots or demo notes.
- Add evaluation criteria and next steps.

Definition of Done:

- One case study is public.
- The case includes problem, process, output, evaluation, and reflection.
- The project card links to a deeper project page or section.

### v0.5 Establish Visual System

Status: Planned

Goal: Create a consistent visual identity around the window theme.

Scope:

- Define color palette.
- Define typography.
- Define spacing rules.
- Define card/window component styles.
- Decide image direction.

Definition of Done:

- The site has a reusable design system foundation.
- New sections can be added without redesigning from scratch.
- Mobile layout remains readable and calm.
